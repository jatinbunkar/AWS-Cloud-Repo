# Asset Management using AWS Config, Native Scripts, and EBS Details

## Steps:

## 1. Enable AWS Config
1. Navigate to the **AWS Config** service in the AWS Management Console.
2. Click on **Get started** (if not already configured).
3. Configure the settings:
   - **Resource Types to Record**: Choose "Record all resources supported in this region."
   - **Delivery Method**: Specify an S3 bucket to store the configuration snapshots.
   - **AWS Config Role**: Use an existing role or create a new one with the required permissions.
4. Click **Next**, review the settings, and enable AWS Config.


## 2. Create a Config Rule for Asset Tracking
1. Navigate to the **Rules** section in AWS Config.
2. Click **Add Rule**.
3. Search for and select a relevant rule (e.g., `required-tags` to ensure assets are tagged).
4. Configure the rule:
   - Provide mandatory tags like `Environment`, `Owner`, etc.
   - Set the rule's trigger frequency (e.g., periodic or configuration change).
5. Click **Save** to enable the rule.

## 3. Validate AWS Config Data
1. Navigate to the **Resources** section in AWS Config.
2. Review the list of tracked resources (EC2 instances, S3 buckets, EBS volumes, etc.).
3. Verify compliance of resources with the Config rules.

## 4. Write a Native Script for Asset Information
1. Open your terminal or connect to an EC2 instance with AWS CLI installed.
2. Use the AWS CLI to list assets:
   - **EC2 Instances**:
     ```bash
     aws ec2 describe-instances --query "Reservations[*].Instances[*].[InstanceId, Tags[0].Value || 'No Tags']" --output table

     ```
   - **S3 Buckets**:
     ```bash
     aws s3api list-buckets --query "Buckets[*].Name" --output table
     
     ```
   - **EBS Volumes**:
     ```bash
     aws ec2 describe-volumes --query "Volumes[*].[VolumeId, Size, State]" --output table

     ```

3. Save the output to a file for further analysis:
   ```bash
   aws ec2 describe-instances --query "Reservations[*].Instances[*].[InstanceId, Tags[0].Value || 'No Tags']" --output tabl > ec2_assets.txt
   aws s3api list-buckets --query "Buckets[*].Name" --output table > s3_assets.txt
   aws ec2 describe-volumes --query "Volumes[*].[VolumeId, Size, State]" --output table > ebs_details.txt
   ```

## 5. Automate Asset Reporting with Native Scripts
   - Write a script to collect asset details periodically. Example:

   ```bash
   #!/bin/bash
   echo "Collecting EC2 Instances..."
   aws ec2 describe-instances --query "Reservations[*].Instances[*].[InstanceId,Tags]" --output table > ec2_assets.txt

   echo "Collecting S3 Buckets..."
   aws s3api list-buckets --query "Buckets[*].Name" --output table > s3_assets.txt

   echo "Collecting EBS Volumes..."
   aws ec2 describe-volumes --query "Volumes[*].[VolumeId,Size,State,Tags]" --output table > ebs_details.txt

   echo "Asset details collected successfully!"
   ```

   - Make the script executable:

   ```bash
   chmod +x asset_management.sh
   ```

   - Schedule the script using cron (Linux) or Task Scheduler (Windows).

  ## 6. Monitor Asset Compliance

- Use the **Compliance Dashboard** in AWS Config to track:
  - Non-compliant resources.
  - Resources missing required tags or configurations.
- Generate reports by exporting the data for offline analysis.

## 7. Clean Up Resources

- Disable AWS Config if no longer required:
  - Go to the **Settings** section and click **Delete Configuration Recorder**.
- Remove the S3 bucket or clear data if it was created specifically for the lab.
- Delete any rules or roles created during the lab.

## Summary

This lab refers to the process of setting up AWS Config for asset management, tracking resources in your AWS environment, ensuring compliance through rules, collecting and automating asset information using scripts, and managing non-compliant resources. It aims to provide a comprehensive way to automate, monitor, and report on AWS resources and configurations.
